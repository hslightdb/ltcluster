C.1.1. CentOS repositories
C.1.2. CentOS package details

Currently, ltcluster RPM packages are provided for versions 6.x and 7.x of CentOS. These should also work on matching versions of Red Hat Enterprise Linux, Scientific Linux and Oracle Enterprise Linux; together with CentOS, these are the same RedHat-based distributions for which the main community project (PGDG) provides packages (see the lt_ RPM Building Project page for details).

Note these ltcluster RPM packages are not designed to work with SuSE/OpenSuSE.

Note

ltcluster packages are designed to be compatible with community-provided lt_ packages. They may not work with vendor-specific packages such as those provided by RedHat for RHEL customers, as the filesystem layout may be different to the community RPMs. Please contact your support vendor for assistance.

C.1.1. CentOS repositories

ltcluster packages are available from the public Hundsun LightDB repository, and also the lt_ community repository. The Hundsun LightDB repository is updated immediately after each ltcluster release.

C.1.2. CentOS package details

The two tables below list relevant information, paths, commands etc. for the ltcluster packages on CentOS 7 (with systemd) and CentOS 6 (no systemd). Substitute the appropriate lt_ major version number for your installation.

Note

For lt_ 9.6 and lower, the CentOS packages use a mixture of 9.6 and 96 in various places to designate the major version; e.g. the package name is ltcluster96, but the binary directory is /var/lib/pgsql/9.6/data.

From lt_ 10, the first part of the version number (e.g. 10) is the major version, so there is more consistency in file/path/package naming (package ltcluster10, binary directory /var/lib/pgsql/10/data).

Table C.3. CentOS 7 packages

Package name example:ltcluster11-4.4.0-1.rhel7.x86_64
Metapackage:(none)
Installation command:yum install ltcluster11
Binary location:/usr/pgsql-11/bin
ltcluster in default path:NO
Configuration file location:/etc/ltcluster/11/ltcluster.conf
Data directory:/var/lib/pgsql/11/data
ltclusterd service command:systemctl [start|stop|restart|reload] ltcluster11
ltclusterd service file location:/usr/lib/systemd/system/ltcluster11.service
ltclusterd log file location:(not specified by package; set in ltcluster.conf)
