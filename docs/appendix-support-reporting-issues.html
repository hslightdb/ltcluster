D.1. Reporting Issues

When asking questions or reporting issues, it is extremely helpful if the following information is included:

  • LightDB version
  • ltcluster version
  • How was ltcluster installed? From source? From packages? If so from which repository?
  • repmpgr.conf files (suitably anonymized if necessary)
  • Contents of the ltcluster.nodes table (suitably anonymized if necessary)
  • LightDB 11 and earlier: contents of the recovery.conf file (suitably anonymized if necessary).
  • LightDB 12 and later: contents of the postgresql.auto.conf file (suitably anonymized if necessary), and whether or not the LightDB data directory contains the files standby.signal and/or recovery.signal.

If issues are encountered with a ltcluster client command, please provide the output of that command executed with the options -LDEBUG --verbose, which will ensure ltcluster emits the maximum level of logging output.

If issues are encountered with ltclusterd, please provide relevant extracts from the ltcluster log files and if possible the LightDB log itself. Please ensure these logs do not contain any confidential data.

In all cases it is extremely useful to receive as much detail as possible on how to reliably reproduce an issue.
