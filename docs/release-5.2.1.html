A.1. Release 5.2.1

A.1.1. Improvements
A.1.2. Bug fixes

Mon 7 December, 2020

ltcluster 5.2.1 is a minor release.

A.1.1. Improvements

  • ltcluster standby clone: option --recovery-min-apply-delay added, overriding any setting present in ltcluster.conf.

A.1.2. Bug fixes

  • Configuration: fix parsing of replication_type configuration parameter. GitHub #672.

  • ltcluster standby clone: handle case where postgresql.auto.conf is absent on the source node.

  • ltcluster standby clone: in LightDB 11 and later, an existing data directory's permissions will not be changed to 0700 if they are already set to 0750.

  • ltclusterd: prevent termination when local node not available and standby_disconnect_on_failover is set. GitHub #675.

  • ltclusterd: ensure reconnect_interval is correctly handled. GitHub #673.

  • ltcluster witness --help: fix witness unregister description. GitHub #676.
