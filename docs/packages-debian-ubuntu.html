C.2.2. Debian/Ubuntu package details

ltcluster .deb packages are provided by Hundsun LightDB as well as the lt_ Community APT repository, and are available for each community-supported lt_ version, currently supported Debian releases, and currently supported Ubuntu LTS releases.

C.2.2. Debian/Ubuntu package details

The table below lists relevant information, paths, commands etc. for the ltcluster packages on Debian 9.x ("Stretch"). Substitute the appropriate lt_ major version number for your installation.

See also Section 13.2.2 for some specifics related to configuring the ltclusterd daemon.

Table C.7. Debian 9.x packages

Package name example:lightdb-11-ltcluster
Metapackage:ltcluster-common
Installation command:apt-get install lightdb-11-ltcluster
Binary location:/usr/lib/lightdb/11/bin
ltcluster in default path:Yes (via wrapper script /usr/bin/ltcluster)
Configuration file location:(not set by package)
Data directory:/var/lib/lightdb/11/main
lt_ service command:systemctl [start|stop|restart|reload] postgresql@11-main
ltclusterd service command:systemctl [start|stop|restart|reload] ltclusterd
ltclusterd service file location:/etc/init.d/ltclusterd (defaults in: /etc/defaults/ltclusterd)
ltclusterd log file location:(not specified by package; set in ltcluster.conf)

Note

When using Debian packages, instead of using the systemd service command directly, it's recommended to execute pg_ctlcluster (as root, either directly or via sudo), e.g.: 

            pg_ctlcluster 11 main [start|stop|restart|reload]

For pre-systemd systems, pg_ctlcluster can be executed directly by the lightdb user.
