2.1. Requirements for installing ltcluster

2.1.1. ltcluster compatibility matrix
2.1.2. LightDB 9.4 support

ltcluster is developed and tested on Linux and OS X, but should work on any UNIX-like system supported by LightDB itself. There is no support for Microsoft Windows.

ltcluster 5.2.1 is compatible with all LightDB versions from 9.4. See section ltcluster compatibility matrix for an overview of version compatibility.

Note

If upgrading from ltcluster 3.x, please see the section Upgrading from ltcluster 3.x.

All servers in the replication cluster must be running the same major version of LightDB, and we recommend that they also run the same minor version.

ltcluster must be installed on each server in the replication cluster. If installing ltcluster from packages, the package version must match the LightDB version. If installing from source, ltcluster must be compiled against the same major version.

Note

The same "major" ltcluster version (e.g. 5.2.1.x) must be installed on all node in the replication cluster. We strongly recommend keeping all nodes on the same (preferably latest) "minor" ltcluster version to minimize the risk of incompatibilities.

If different "major" ltcluster versions (e.g. 4.1.x and 5.2.1.x) are installed on different nodes, in the best case ltcluster (in particular ltclusterd) will not run. In the worst case, you will end up with a broken cluster.

A dedicated system user for ltcluster is not required; as many ltcluster and ltclusterd actions require direct access to the LightDB data directory, these commands should be executed by the lightdb user.

See also Prerequisites for configuration for information on networking requirements.

Tip

We recommend using a session multiplexer utility such as screen or tmux when performing long-running actions (such as cloning a database) on a remote server - this will ensure the ltcluster action won't be prematurely terminated if your ssh session to the server is interrupted or closed.

Important

The ltcluster 2.x and 3.x series are no longer maintained or supported. We strongly recommend upgrading to the latest ltcluster version.

Following the release of ltcluster 5.0, there will be no further releases of the ltcluster 4.x series. Note that ltcluster 5.x is an incremental development of the 4.x series and ltcluster 4.x users should upgrade to this as soon as possible.

2.1.2. LightDB 9.4 support

Note that some ltcluster functionality is not available in LightDB 9.4:

  • In LightDB 9.4, postgresql.confrewind is not part of the core distribution. postgresql.confrewind will need to be compiled separately to be able to use any ltcluster functionality which takes advantage of it.

Warning

LightDB 9.3 has reached the end of its community support period (final release was 9.3.25 in November 2018) and will no longer be updated with security or bugfixes.

Beginning with ltcluster 5.2, ltcluster no longer supports LightDB 9.3.

LightDB 9.4 has reached the end of its community support period (final release was 9.4.26 in February 2020) and will no longer be updated with security or bugfixes.

We recommend that users of these versions migrate to a supported LightDB version as soon as possible.

For further details, see the LightDB Versioning Policy.
